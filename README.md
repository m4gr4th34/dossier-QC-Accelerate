## The argument in one move

A qubit is powerful and fragile for the same reason: it holds a blend of 0 and 1 that
leaks into the world at the slightest disturbance. The field's answer, quantum error
correction, works but is hungry — about a thousand physical qubits per reliable one. This
dossier climbs a four-rung ladder, each rung the same idea at a different depth: **hide the
information where the noise cannot reach it.**

1. **Engineering layer** — AI decoders and below-threshold surface codes make today's machine better.
2. **Designed qubits** — let AI design the qubit's circuit or screen new materials: a *different* qubit.
3. **Self-correcting matter** — store the qubit passively, protected by global structure and an energy gap.
4. **The floor** — engineer the equation of motion so a working qubit is its *steady state*, and errors heal themselves.

At the floor we build a ~60-line cat-qubit simulation that reproduces the exponential noise
bias of dissipative cats from first principles; document a reward-hacking failure of a naive
discovery metric, *kept* rather than hidden; and give a Knill–Laflamme proof that the
four-component cat turns single-photon loss into a detectable, asymptotically correctable
Z₄ syndrome (with k = 4 the minimal such code).

Chapter 2 then asks the next question: with a place to *store* a protected qubit, can you *operate* on one — run a logic gate — without breaking the protection? This is a third thing an AI search can do, past re-deriving and discovering: *scope* a frontier honestly.

With a scorecard built so it cannot be reward-hacked, a search maps the bias-preserving gate's κ₁/κ₂ frontier with more freedom than any human design, seeded from the best human gate — and it returns a rigorous *kept negative*: the hand-designed gate already sits at the optimum of this control space, and handed a bias-breaking shortcut, the search drove it to zero on its own. A clean “no,” and a precise map of where the wall sits.

**No novel physics is claimed.** The underlying results are prior work (Mirrahimi 2014; the
self-correcting-memory literature; reinforcement-learning code-discovery efforts). The
contribution is methodological: a reproducible, honestly-labelled synthesis in which a
negative result was kept and every number is regenerated by code.

## Reproduce it

```bash
pip install numpy matplotlib
cd verification/floor-sim
python figures.py     # floor sim: exponential noise bias, regenerates fig1-fig2 (self-tests on run)
python kl_proof.py    # Knill-Laflamme correctability: 2-cat 0%, 3-cat 50%, 4-cat 100%; k=4 minimal
python figures3.py    # regenerates the KL proof figure (fig5)
```

Pure NumPy, no quantum libraries. The scripts self-test on every run; if a number in the
paper ever disagrees with the code, the code wins.
