Citation audit

In a survey, the citations carry the verification weight: every avenue stands on its sources, so every source is checked against the primary work or triangulated through three or more independent citing works. Interested-party sources (vendor roadmaps, press releases) are marked REPORTED, not verified — the provenance is the point. ADVERSARIAL REVIEW

Red-team findings

A hostile pass over the finished survey, published with it. Resolved findings were fixed before launch; open findings are stated as caveats in the avenues themselves.

RESOLVEDReward-hacking of the discovery metric
A blank search "improved" the bit-flip score by tilting the dissipator so single-photon loss drained one logical state into the other — the qubit was collapsing, not protecting. Caught by adding convergence and balance guards to the metric; kept as open challenge C10 rather than hidden.
RESOLVEDCitation year/volume errors corrected against primary sources
Three references carried wrong values: AlphaQubit (Nature 634 → 635), the below-threshold result (2024 → 2025, vol. 638), and Gertler et al. (2020 → 2021, vol. 590). Caught by re-verifying each against the primary source before finalizing the manuscript; corrected.
RESOLVEDFigure-reproducibility bug in fig3.py
The reward-hacking figure's script hard-coded a sandbox output path and aborted before its in-tree save, so an external reader could not reproduce it. Caught by running the figures from a clean checkout; fixed to resolve the output directory from __file__ so it reproduces from any working directory.
RESOLVEDOver-claim in the §4 discovery framing
A draft asserted the search "recovers the cat as the best code in its family" — implying autonomous discovery. Scoped down to "validates the measurement, not a discovery," and the gap-only metric was shown insufficient on its own.
RESOLVEDNovelty over-reach retracted after prior-art search
Prior-art reconnaissance established that "fault tolerance as a phase of matter" and Knill–Laflamme-objective code search are both prior art. The dossier now claims no novel physics (ledger C03); its contribution is the reproducible, honestly-negative synthesis.
RESOLVEDLedger/paper desynchronization
The paper referenced ledger IDs (C08, C10, C11, C12) that did not yet exist in claim_ledger.csv — it still held the four template placeholder rows. Caught during a release-prep pass; the ledger was authored to match every ID the editions reference.
OPENAutonomous discovery is unverified (C10 / C04)
Whether an automated search can discover a loss-correcting code unaided — rather than reproduce the known cat — is unproven. Posted as an open challenge with a dated signpost (C04: a reproduced result beyond the cat family by 2031).
OPENApproximate-code caveats not yet bounded in-repo
The Knill–Laflamme correctability of the four-component cat is exact only in the large-α limit; at small cat size the coherent-state lobes overlap. This is stated honestly in the text but not yet quantified with an explicit error bound in the repository.
THE METHOD

The Open Dossier pipeline

The architecture that produced this record — reusable for every subsequent dossier.

  1. Prior-art reconnaissanceExtract candidate novelty claims; search each one, its components, and combinations; produce the established / adjacent / open table.
  2. Claim ledgerEvery claim typed — CITE, NUM, DERIV, NOVEL, EST, FORECAST — routed to its mandated verifier. A FORECAST's verifier is its dated, falsifiable signpost.
  3. DraftingSurvey written against the open ledger. Reproducibility section and AI-assistance disclosure are mandatory.
  4. Mechanical verificationConsistency checks recomputed by script; every citation checked. Fail = fix the paper, never the tolerance.
  5. Adversarial reviewFresh AI sessions briefed as hostile referees. Findings published RESOLVED or OPEN.
  6. Human reviewConsistency and reality checks on end results. Per-claim waiver allowed — at exactly one price: the label tells the truth.
  7. PublicationRepo live → GitHub Pages → Zenodo DOI (auto) → OpenTimestamps (auto) → community review via issues.
